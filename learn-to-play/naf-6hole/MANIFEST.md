# MANIFEST — A-tuned 6-hole NAF starter songbook

Every file written into this folder by the v0.1 `sheet-music` skill,
with a one-line purpose. Generated against `naf-6hole` (registry id) /
A-tuned NAF / range A4–D6 / A minor pentatonic.

## Top-level files

| File | Purpose |
|---|---|
| `README.md` | Entry point — repertoire list, practice path, file-format key. |
| `songsheet.pdf` | One-page printable summary of the whole songbook. |
| `fingering-charts.svg` | Combined fingering chart for every NAF pitch (A4, C5, D5, E5, G5, A5, C6, D6). |
| `MANIFEST.md` | This file. |
| `EVAL.md` | Honest evaluation of v0.1 skill behavior on the production-ready NAF path. |

## `01-easy/`

Three slow, beginner-friendly PD tunes pentatonicised for A-NAF. Each
folder contains the same set of files.

### `amazing-grace/`
- `tune.abc` — Newton 1779 / "New Britain" 1835 hymn, transposed up an octave from canonical G to fit A-NAF; B5→C6 substitution.
- `tune.musicxml` — Same, in MusicXML for DAWs.
- `tune.mid` — Playable MIDI.
- `tune.pdf` — Per-tune one-page songsheet.
- `tune-fingering.svg` — Fingering chart for the 6 distinct pitches in this tune.
- `notes.md` — Provenance, arrangement explanation, practice tip.
- `render-summary.json` — Pipeline-stage status (which renderers ran / were skipped).

### `star-of-the-county-down/`
- `tune.abc` — Trad. Irish modal melody from the "Dives and Lazarus" / Petrie-1855 family, pentatonicised from A dorian.
- `tune.musicxml`, `tune.mid`, `tune.pdf`, `tune-fingering.svg`, `notes.md`, `render-summary.json` — same purposes as above.

### `wondrous-love/`
- `tune.abc` — Trad. Appalachian (1811 lyrics / 1835 melody), pentatonicised from A dorian.
- `tune.musicxml`, `tune.mid`, `tune.pdf`, `tune-fingering.svg`, `notes.md`, `render-summary.json` — same purposes.

## Files NOT generated (and why)

| Missing file pattern | Reason | How to fix |
|---|---|---|
| `*/tune.ly` | LilyPond binary not on PATH on this machine. music21's LilyPond converter requires the system binary even when music21 itself is installed. | Install `lilypond` (apt/brew/choco) and re-run `scripts/render_pipeline.py`. |
| `*/tune.svg` | Depends on `tune.ly`. | Same as above. |
| `*/tune.wav` | `fluidsynth` and `timidity` both absent. | Install `fluidsynth` + a soundfont (e.g., FluidR3_GM) per `references/audio-rendering.md`. |
| `*/ableton-prompt.md` | Optional handoff file; not auto-generated by the deposit pipeline (only by `scripts/build_ableton_handoff.py` on demand). | Run `scripts/build_ableton_handoff.py --tune-dir <path> --instrument naf-6hole` if you want one. |

The `render-summary.json` files inside each tune folder document
exactly which stages ran on this generation and which were skipped, so
the "missing" status is auditable rather than mysterious.
